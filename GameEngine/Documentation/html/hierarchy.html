This inheritance list is sorted roughly, but not completely, alphabetically:
[detail level 12]
 CApplicationThis is the Application Class and is the core of the engine where our entities and components will be called from and checked
 ▼CComponentThe component class will be used to store all the components we can add to the entities such as boxcolliders, camera, keyboard, etc
 CBoxColliderBoxCollider class will be used to calculate the collison between objects in the scene, for example: the player and another object in the scene such as a box
 CCameraThis is the Camera class and inherits from Component class, this class is where we create our viewing and projection matrix and initialize our cameras
 CEnvironmentThis is the Environment class which handles the time such as delta time, ensuring no matter the hardware being used all the changes are the same, such as player movement
 CGUIThe GUI/Graphic User interface is responsible for handling buttons, icons, player health display, logo, etc
 CKeyboardKeyboard Class that handles input such as keyboard and mouse by the user
 CMeshrendererThis is thhe Meshrenderer Component which will handle our meshes, materials and display them onto the screen
 CPlayerControlThis is the PlayerControl struct which handles player movement and Inherits from Components
 CTransformThe transform class is inherited from the Component class,and it handles the translation and movements of objects in the scene
 CEntityEntity class is where we store the different types of gameobjects in the scene and checks for the different types of components using templates, then adds the components to the list of components each entity keeps
 ▼Cexception
 CExceptionThis is a struct Exception which is a personalised exception that allows us to type the message we want to output during errors during run time, so users can indentify where errors are happening more easily
 ▼CResourceResource class is the class that stores resources such as Material and Mesh, and passes information to other resources from the Resources Class
 CMaterialThe Material Class will be handling the texture files users want to use for their objects
 CMeshThis is the Mesh class, it is responsible for loading in models into the scene such as .obj files
 CResourcesThis is the Resources class which manages all resources in scene such as mesh and Material
